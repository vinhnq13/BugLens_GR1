# docs/testcases/

**Test Cases and Test Plans**

This folder contains test documentation for BugLens, including:

- Unit test plans for backend-api modules
- Integration test plans for API endpoints
- End-to-end (E2E) test scenarios
- AI model evaluation test cases (accuracy, precision, recall)
- Manual test scripts for browser extension and dashboard

## Test Naming Convention

```
TC-<Module>-<Number>: <Short Description>
Example: TC-ISSUE-001: Submit a new bug report via API
```

> 📝 Test case documents are added progressively during and after feature implementation (Phases 2–5).
